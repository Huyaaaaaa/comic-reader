# Comic Go - 漫画离线浏览系统 需求说明书 V1.0

**文档版本：** 1.0  
**最后更新：** 2026-03-10  
**适用对象：** 产品团队、项目经理、设计团队、测试团队、外部评审、内部 AI

---

## 1. 项目目标

Comic Go 是一个面向桌面端的漫画离线浏览系统，目标是提供“本地优先、离线可用、远端顺序一致、可控缓存、多源容灾”的阅读体验。

核心目标如下：
- 列表页、详情页、阅读页在无网络时仍然可打开
- 页面点击后先进入页面框架，再异步补数据，避免明显等待感
- 本地缓存优先命中，网络只负责补全或更新
- 阅读页优先保证图片回显，后台缓存不能抢占阅读体验
- 远端站点发生更新时，新漫画可以优先进入本地缓存链路

---

## 2. 核心原则

- **本地优先**：查询先读本地，本地缺失时再联网补全
- **离线可用**：网络不可达时不阻断页面进入，只提示当前为离线数据
- **懒加载优先**：列表、详情、阅读都优先进入页面，再按区块补内容
- **远端排序一致**：本地“所有漫画”页的顺序应与远端默认排序保持一致
- **多源可扩展**：支持多个源站，支持故障切换与后续导航页自动导入
- **多账户预留**：V1 以本地默认用户运行，但结构按多用户预留

---

## 3. 缓存层定义

### 3.1 L1 列表元数据

用于列表页快速展示的基础信息，包括但不限于：
- 漫画 ID
- 标题 / 副标题
- 分类
- 评分 / 评分人数
- 远端收藏数
- 远端排序位置

### 3.2 L2 详情扩展数据

用于详情页和增强搜索的扩展信息，包括但不限于：
- 封面与封面本地缓存
- 作者信息（支持多作者）
- 标签信息（支持多标签）
- 图片总数
- 详情补充字段

### 3.3 L3 正文图片

用于阅读页和离线阅读的图片索引与图片文件，包括：
- 图片顺序
- 图片远端地址
- 图片本地缓存
- 文件大小与校验信息

## 4. 页面与功能范围

### 4.1 V1 包含

- 首页总览 Dashboard
- 所有漫画列表页
- 详情页
- 阅读页
- 收藏
- 阅读历史
- 下载管理
- 缓存管理
- 设置中心
- 导入导出
- 内容更新与应用更新
- 多源站管理

### 4.2 V1 暂不包含

## 5. 页面行为要求

### 5.1 列表页

- 每页默认显示 100 条
- 后端接受 1 到 100 的任意页容量
- 前端提供符合排版的预设页容量选项
- 列表点击后应先跳转详情页框架，再异步补全数据
- 列表页应优先使用本地 L1 数据
- 当本地没有足够数据时，允许联网补全
- “所有漫画”总页数来自本地保存的远端快照
- 启动后 30 秒内异步尝试刷新远端总页数与总漫画数

### 5.2 详情页

- 点击后先进入详情页框架，不等待网络返回
- 详情页中的标题、封面、作者、标签、图片数量、同作者作品应支持分块懒加载
- 详情页优先读取本地 L2 数据
- 本地 L2 缺失时再联网补全
- 同作者作品优先本地匹配，再按网页搜索逻辑联网补全
- 同作者结果应尽量与原网页一致

### 5.3 阅读页

- Phase A 先实现长图流模式
- 后续补充左右翻页和上下翻页模式
- 首屏预加载 3 张，滚动预取 5 张
- 阅读页优先保证当前阅读图片加载
- 同一图片远端请求只能发生一次
- 阅读页已经拉到的图片，后台被动缓存应复用该结果，不允许重复向远端拉取
- 阅读进度记录为“图序号 + 图内滚动比例”
- 首次打开默认非全屏，后续跟随上次关闭时的模式

### 5.4 首页总览 Dashboard

V1 保留首页总览，至少包含：
- 缓存统计
- 下载统计
- 最近任务
- 环形图
- 快捷入口
- 实时日志输出框

---

## 6. 搜索与筛选要求

- 搜索范围覆盖标题、作者、标签
- 当本地元数据尚未完整时：
  - 先用本地结果回显一次
  - 提示“正在远程搜索更多”
  - 后台补一次远端搜索并增量回显
- 作者支持多对多
- 标签支持多对多
- 分类使用独立分类集合

---

## 7. 缓存策略要求

V1 的缓存不是单一总模式，而是 `L1 / L2 / L3` 三层独立配置。

### 7.1 L1 配置能力

- `none`
- `passive`
- `active_count`
- `active_all`

L1 的 `passive` 还需要支持触发范围：
- 仅列表触发
- 仅详情触发
- 列表与详情都可触发

### 7.2 L2 配置能力

- `none`
- `passive`
- `active_count`
- `active_all`

### 7.3 L3 配置能力

- `none`
- `passive`
- `active_count`
- `active_all`

### 7.4 默认缓存预设

默认采用“全部被动缓存”：
- `L1 passive(list_and_detail)`
- `L2 passive`
- `L3 passive`

### 7.5 常用预设

前端至少提供以下常用预设：
1. 默认全部被动缓存
2. 主动缓存定量 L1 + L2
3. 被动缓存详情页 L1 + L2
4. 列表被动 L1 + 详情被动 L2
5. 主动全量 L1 + 详情被动 L2
6. 主动全量 L1 + 主动定量 L2
7. 主动全量 L1 + L2

### 7.6 数量配置

- L1、L2、L3 的主动缓存数量彼此独立
- 不共享同一个 N

### 7.7 收藏的特殊规则

- 收藏时可选择“离线可读”
- 一旦选择离线可读，该漫画允许创建独立完整缓存任务
- 不受全局默认缓存策略阻断

---

## 8. 封面缓存要求

- 封面缓存限制支持两种模式：数量限制 / 容量限制
- 默认值：1000 张 或 1GB
- 淘汰策略支持用户选择：`LRU` / `FIFO`
- 收藏漫画封面在取消收藏前永久保留
- 历史记录中的漫画封面不参与保护

---

## 9. 多源站要求

### 9.1 基本能力

- 默认不内置任何源站地址
- 所有源站均由用户手动填写
- 允许后续开发“导航页导入”：用户填写一个导航页地址，系统自动解析并保存其中可用源站
- 支持手动调整源站优先级
- 支持手动测速
- 支持自动故障切换

### 9.2 默认调度策略

- 默认心跳检测间隔：60 分钟
- 单次请求若当前源站超时，应先提示超时
- 单次请求默认自动重试 3 次
- 连续 3 次失败后，标记该源站当前不可用
- 一旦当前源站在本次请求中被判定不可用，立即临时切换到下一个可用源站继续请求
- 后续心跳成功后，可重新恢复到可用池

---

## 10. 更新要求

### 10.1 内容更新

支持三种模式：
- 手动检查
- 启动时检查
- 定时检查

默认：
- 启动后允许异步检查
- 定时间隔默认 30 分钟

### 10.2 应用更新

支持两种模式：
- 手动检查
- 启动时检查

默认：
- 自动检查开启
- 自动安装关闭

---

## 11. 下载与离线要求

- 下载管理区分“进行中”和“已完成”两类列表
- 进行中列表包含：排队、下载中、校验中、暂停、失败等状态
- 下载前必须做磁盘空间预检查
- 下载中继续检测磁盘空间，不足时自动暂停并提示
- 离线状态下应允许浏览已缓存的列表、详情和阅读内容
- 若页面没有本地数据，也应先进入页面并显示空态提示，而不是阻断跳转

---

## 12. 收藏、历史与导出导入要求

### 12.1 收藏

- 详情页和阅读页都可收藏
- 收藏列表支持继续阅读
- 收藏离线可读时，需要保证该漫画可本地完整浏览

### 12.2 阅读历史

- 记录阅读时间
- 记录阅读位置
- 支持继续阅读

### 12.3 导出范围

V1 至少支持：
- 单部漫画导出
- 所选已完全缓存漫画导出
- 全部已缓存漫画导出
- 所有封面导出
- 所有图片导出

### 12.4 导入策略

V1 支持：
- 覆盖导入
- 仅导入缺失
- 取消

---

## 13. 实时反馈要求

V1 使用 `SSE` 提供实时反馈，至少覆盖：
- 下载进度
- 缓存任务进度
- 实时日志输出

目标是让用户能明确看到软件当前在做什么，而不是依赖切换页面后才刷新。

---

## 14. 默认参数总表

- 列表每页默认：100
- 阅读首屏预加载：3
- 阅读滚动预取：5
- 封面缓存默认上限：1000 张 / 1GB
- 头部扫描默认页数：5
- 防封禁默认档位：safe
- 内容更新默认间隔：30 分钟
- 默认缓存预设：全部被动缓存
- 多源站心跳默认间隔：60 分钟
- 单次拉取自动重试：3 次
- 连续失败 3 次标记当前源站不可用

---

## 15. 非目标

V1 不要求实现：
- 云同步
- 拼音搜索
- 阅读缩放
- 主题切换
- 本地评分
