## 0. 2026-03-10 最新决议补充

### 0.1 已确认事项

- **源站管理**：V1 支持多个源站，默认不内置任何地址，全部由用户手动填写。后续增加"导航页导入"能力，用户填写一个导航页地址后，系统自动解析并保存其中全部可用源站地址。
- **远端总数快照**：启动时不阻塞主流程；应用启动后 30 秒内异步尝试获取远端总页数与总漫画数，并将成功结果持久化到本地。若本次获取失败，则继续使用上一次成功快照，并提示数据可能不是最新。
- **缓存层定义**：L1 为列表元数据，L2 为详情扩展元数据，L3 为正文图片索引与图片文件。
- **首页总览**：V1 保留 Dashboard，包含缓存统计、下载统计、最近任务、实时日志输出框、环形图与快捷入口。
- **阅读进度**：长图流模式记录到“图序号 + 图内滚动比例”；该结构同时服务于继续阅读与离线恢复。
- **封面淘汰**：封面缓存淘汰策略由用户在设置中选择 `LRU` 或 `FIFO`；收藏漫画封面在取消收藏前永久保留；历史记录不参与保护。
- **更新模式**：内容更新支持 `manual` / `startup` / `interval`；应用更新支持 `manual` / `startup`。
- **搜索范围**：本地搜索覆盖标题、作者、标签；若本地元数据尚未完整，前端先回显本地结果并提示“正在远程搜索更多”，后台补一次远端搜索并增量回显。
- **作者匹配**：优先按远端作者 ID 精确匹配；若远端没有稳定 ID，则按 `normalized_name` 严格匹配；同作者结果应尽量与原网页搜索回显一致。
- **导出范围**：除单漫画外，V1 需要支持“筛选部分本地已完全缓存漫画导出”与“全部已缓存漫画导出”两类范围。
- **实时事件流**：V1 使用 `SSE` 推送下载进度、缓存任务进度与实时日志，避免仅靠轮询。
- **多账户预留**：架构直接按多用户适配设计；未登录时使用默认本地用户槽位。
- **默认缓存预设**：默认采用"全部被动缓存"，即 `L1 passive + L2 passive + L3 passive`。
- **主动缓存数量**：L1 / L2 / L3 的主动缓存数量彼此独立，分别配置。
- **源站阈值默认值**：心跳检测默认间隔为 60 分钟；单次请求若当前源站不可达则先提示超时并自动重试，连续 3 次失败后标记该源站当前不可用，并在本次请求内立即切换到下一个可用源站继续。

## 1. 项目概述

### 1.1 项目目标

提供一个"本地优先、离线可用"的漫画浏览系统，保证列表页、详情页、阅读页在无网络时仍可打开并可用。

### 1.2 核心定位

- **本地优先**：接口优先返回本地数据，网络只做补全
- **离线可用**：所有核心功能离线可用
- **渐进式缓存**：L1 列表 → L2 详情 → L3 图片
- **远端排序一致**：新漫画始终显示在列表最前
- **多源可扩展**：V1 支持用户手填多个源站，并预留导航页自动发现能力

### 缓存层定义

- **L1 列表元数据**：用于列表页快速展示的基础字段，例如 `comic_id`、标题、副标题、评分、评分人数、远端收藏数、分类、远端排序索引等
- **L2 详情扩展数据**：用于详情页与搜索增强的扩展字段，例如封面、本地封面路径、作者、多作者关联、标签、图片总数、详情页补充描述等
- **L3 正文图片数据**：阅读页使用的图片索引与图片文件本体，包括图片 URL、排序、本地相对路径、文件大小与校验信息

### 1.3 功能范围（V1）

**包含：**
- 列表页、详情页、阅读页
- 首页总览（Dashboard）
- 收藏、阅读历史
- 下载管理
- 缓存管理
- 设置中心
- 导入导出
- 更新中心（内容更新 + 应用更新）
- 多源站管理

**不包含：**
- 本地评分
- 多端账号同步（仅预留结构）
- 阅读缩放（V1 延后）
- 拼音搜索（V1 延后）
- 主题切换（V1 延后）

### 1.4 技术栈

- **后端**：Go 1.21+ + Gin + GORM + SQLite
- **前端**：React 18 + TypeScript + Vite + Tailwind CSS
- **部署**：可嵌入 Electron 打包为桌面应用

**触发器：自动维护 images_local**

```sql
-- 插入图片时 +1
CREATE TRIGGER IF NOT EXISTS trg_images_local_inc
AFTER INSERT ON comic_images
WHEN TRIM(COALESCE(NEW.local_rel_path, '')) != ''
BEGIN
  INSERT OR IGNORE INTO comic_cache_state (comic_id) VALUES (NEW.comic_id);
  UPDATE comic_cache_state
  SET images_local = images_local + 1,
      updated_at = datetime('now')
  WHERE comic_id = NEW.comic_id;
END;

-- 删除图片时 -1（防负数）
CREATE TRIGGER IF NOT EXISTS trg_images_local_dec
AFTER DELETE ON comic_images
WHEN TRIM(COALESCE(OLD.local_rel_path, '')) != ''
BEGIN
  UPDATE comic_cache_state
  SET images_local = MAX(0, images_local - 1),
      updated_at = datetime('now')
  WHERE comic_id = OLD.comic_id;
END;

-- 更新图片路径时
CREATE TRIGGER IF NOT EXISTS trg_images_local_upd
AFTER UPDATE OF local_rel_path ON comic_images
WHEN (TRIM(COALESCE(OLD.local_rel_path, '')) = '' AND TRIM(COALESCE(NEW.local_rel_path, '')) != '')
   OR (TRIM(COALESCE(OLD.local_rel_path, '')) != '' AND TRIM(COALESCE(NEW.local_rel_path, '')) = '')
BEGIN
  UPDATE comic_cache_state
  SET images_local = images_local + CASE
      WHEN TRIM(COALESCE(NEW.local_rel_path, '')) != '' THEN 1
      ELSE -1
    END,
    updated_at = datetime('now')
  WHERE comic_id = NEW.comic_id;
END;
```

## 5. 核心业务规则

### 5.1 下载任务状态机

```
状态定义：
- queued: 排队中
- downloading: 下载中
- verifying: 校验中
- paused: 已暂停
- failed: 失败
- completed: 已完成
- canceled: 已取消

允许的状态迁移：
┌─────────┐
│ queued  │──────────────────────────────┐
└─────────┘                              │
     │                                   │
     ↓                                   │
┌─────────────┐                          │
│ downloading │──────────────────────┐   │
└─────────────┘                      │   │
     │                               │   │
     ↓                               │   │
┌───────────┐                        │   │
│ verifying │                        │   │
└───────────┘                        │   │
     │                               │   │
     ↓                               │   │
┌───────────┐                        │   │
│ completed │                        │   │
└───────────┘                        │   │
                                     │   │
┌────────┐                           │   │
│ paused │←─────────────────────────┤   │
└────────┘                           │   │
     │                               │   │
     └───────────────────────────────┘   │
                                         │
┌────────┐                               │
│ failed │←─────────────────────────────┤
└────────┘                               │
     │                                   │
     │ (retry)                           │
     └───────────────────────────────────┘
     
┌──────────┐
│ canceled │←─────────────────────────────┐
└──────────┘                              │
     ↑                                    │
     └────────────────────────────────────┘
     (从任意非 completed 状态可取消)

具体规则：
1. queued → downloading (开始下载)
2. downloading → verifying (下载完成，开始校验)
3. verifying → completed (校验通过)
4. queued/downloading → paused (用户暂停)
5. paused → queued (用户恢复)
6. downloading/verifying → failed (下载/校验失败)
7. failed → queued (用户重试)
8. queued/downloading/verifying/paused/failed → canceled (用户取消)
```

### 5.1A 三层缓存策略

V1 不再使用单一 `cache_mode` 统管全部缓存，而是改为 **L1 / L2 / L3 三层独立配置**，前端可在设置页提供“常用预设 + 高级自定义”两种入口。

**L1 列表元数据模式：**
- `none`：不落盘缓存 L1，列表数据完全依赖在线获取
- `passive`：按被动触发条件缓存 L1
- `active_count`：主动缓存前 N 部漫画的 L1
- `active_all`：主动缓存全部漫画的 L1

**L1 被动触发范围：**
- `list_only`：仅浏览列表页时缓存该页出现的漫画 L1
- `detail_only`：仅进入详情页时缓存该漫画 L1
- `list_and_detail`：浏览列表或进入详情页时都可触发该漫画 L1 缓存

**L2 详情扩展元数据模式：**
- `none`：不落盘缓存 L2
- `passive`：进入详情页时缓存该漫画 L2
- `active_count`：主动缓存前 N 部漫画的 L2
- `active_all`：主动缓存全部漫画的 L2

**L3 正文图片模式：**
- `none`：不主动或被动落盘正文图片
- `passive`：阅读页按阅读进度被动缓存图片
- `active_count`：主动缓存前 N 部漫画的正文图片
- `active_all`：主动缓存全部漫画正文图片

**V1 常用预设应至少包含以下组合：**
1. 默认全部被动缓存：`L1 passive(list_and_detail) + L2 passive + L3 passive`
2. 主动缓存定量 L1+L2：`L1 active_count + L2 active_count + L3 none`
3. 被动缓存详情页 L1+L2：`L1 passive(detail_only) + L2 passive + L3 none`
4. 列表被动 L1 + 详情被动 L2：`L1 passive(list_only) + L2 passive + L3 none`
5. 主动全量 L1 + 详情被动 L2：`L1 active_all + L2 passive + L3 none`
6. 主动全量 L1 + 主动定量 L2：`L1 active_all + L2 active_count + L3 none`
7. 主动全量 L1+L2：`L1 active_all + L2 active_all + L3 none`

**正文图片独立控制：**
- 默认配置下，L3 采用 `passive`，阅读到哪里缓存到哪里
- L1 / L2 / L3 的 `active_count` 数量分别独立配置，不共享同一个 N
- 无论 L1/L2 使用哪一种模式，L3 都可以单独选择 `none` / `passive` / `active_count` / `active_all`
- 阅读体验优先于后台正文下载；同一图片远端请求只能发生一次，阅读回显与后台缓存共用同一结果
- 收藏且勾选离线可读时，可为单漫画创建独立完整缓存任务，不受全局 L1/L2/L3 默认策略阻断

### 5.1B 多源站调度默认策略

- **默认心跳间隔**：60 分钟
- **手动测速**：用户可在设置页手动对全部源站执行一次测速与可用性检查
- **请求超时提示**：单次拉取若超时，前端立即提示当前源站请求超时
- **自动重试**：单次请求默认自动重试 3 次
- **不可用判定**：连续 3 次失败后，标记该源站当前不可用
- **自动故障切换**：一旦当前源站在本次请求中被判定不可用，立即临时切换到下一个可用源站继续请求
- **失败作用域**：源站不可用标记不阻断后续心跳恢复；后续心跳成功后可重新进入可用池
- **源站优先级**：支持手动调整优先级，默认按用户排序依次尝试

### 5.6 阅读页与下载器去重策略

**singleflight 实现：**
```go
var imageGroup singleflight.Group

func FetchImage(comicID int, sort int, url string) ([]byte, error) {
    // 1. 先查本地
    localPath := getLocalImagePath(comicID, sort)
    if localPath != "" && fileExists(localPath) {
        return readFile(localPath)
    }
    
    // 2. singleflight 去重下载
    key := ImageCacheKey(url)
    v, err, _ := imageGroup.Do(key, func() (interface{}, error) {
        data, err := downloadImage(url)
        if err != nil {
            return nil, err
        }
        
        localPath := saveImage(comicID, sort, data)
        updateImageLocalPath(comicID, sort, localPath)
        
        return data, nil
    })
    
    return v.([]byte), err
}
```

### 5.7 磁盘空间策略

func CheckDiskSpace(estimatedSize int64) error {
    var stat syscall.Statfs_t
    syscall.Statfs(cacheDir, &stat)
    availableSpace := int64(stat.Bavail * uint64(stat.Bsize))
    
    reservedSpace := int64(1 * 1024 * 1024 * 1024) // 1GB
    
    if availableSpace < estimatedSize + reservedSpace {
        return fmt.Errorf("磁盘空间不足：需要 %dMB，剩余 %dMB",

    return nil
}

### 5.11 离线能力保证

**核心原则：**
1. 所有页面可打开（不阻断）
2. 优先返回本地数据
3. 网络失败仅提示，不报错
4. 返回数据带 `meta.offline` 标识

**实现要点：**
```go
func GetComics(c *gin.Context) {
    // 1. 先查本地
    localComics := getLocalComics()
    
    // 2. 检查网络
    if !isOnline() {
        c.JSON(200, ApiResponse{
            Data: localComics,
            Meta: Meta{
                Offline: true,
                Stale:   true,
                Source:  "local_cache",
                Message: "当前离线，显示本地缓存数据",
            },
        })
        return
    }
    
    // 3. 在线补全（超时 2 秒）
    ctx, cancel := context.WithTimeout(c.Request.Context(), 2*time.Second)
    defer cancel()
    
    remoteComics, err := fetchRemoteComics(ctx)
    if err != nil {
        // 超时或失败，返回本地数据
        c.JSON(200, ApiResponse{
            Data: localComics,
            Meta: Meta{
                Offline: false,
                Stale:   true,
                Source:  "local_cache",
                Message: "网络请求超时，显示本地缓存数据",
            },
        })
        return
    }
    
    // 4. 返回远端数据
    c.JSON(200, ApiResponse{
        Data: remoteComics,
        Meta: Meta{
            Offline: false,
            Stale:   false,
            Source:  "remote",
        },
    })
}
```

## 8. 部署指南

### 8.1 开发环境搭建

#### 后端环境

**系统要求：**
- Go 1.21+
- SQLite 3.35+

**安装步骤：**
```bash
# 1. 克隆仓库
git clone <仓库地址>
cd comic-go/backend

# 2. 安装依赖
go mod download

# 3. 初始化数据库
sqlite3 comics_go.db < schema.sql

# 4. 启动开发服务器
go run main.go
```

**配置环境变量：**
```bash
export PORT=8080
export DB_SOURCE=/path/to/comics_go.db
export CACHE_DIR=/path/to/cache
export JWT_SECRET=your-secret-key
```

---

#### 前端环境

**系统要求：**
- Node.js 18+
- npm 或 yarn

**安装步骤：**
```bash
# 1. 进入前端目录
cd comic-go/frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

**配置 API 地址：**
```typescript
// src/config.ts
export const API_BASE_URL = '<由部署环境注入>';
```

---

### 8.2 生产环境部署

#### 后端部署

**编译：**
```bash
cd backend
go build -o comic-go-server main.go
```

**运行：**
```bash
./comic-go-server
```

**使用 systemd 管理：**
```ini
# /etc/systemd/system/comic-go.service
[Unit]
Description=Comic Go Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/comic-go
ExecStart=/opt/comic-go/comic-go-server
Restart=on-failure
Environment="PORT=8080"
Environment="DB_SOURCE=/var/lib/comic-go/comics.db"

[Install]
WantedBy=multi-user.target
```

**启动服务：**
```bash
sudo systemctl enable comic-go
sudo systemctl start comic-go
```

---

#### 前端部署

**编译：**
```bash
cd frontend
npm run build
```

**部署到 Nginx：**
```nginx
server {
    listen 80;
    server_name <部署域名>;
    
    root /var/www/comic-go/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass <后端服务上游>;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

---

npm install electron electron-builder --save-dev

### 9.3 常见问题

**Q1: 如何清空所有缓存？**
```sql
DELETE FROM comic_cache_state;
DELETE FROM comic_images WHERE local_rel_path != '';
-- 手动删除缓存目录文件
```

**Q2: 如何重置 sort_key？**
```sql
WITH ranked AS (
  SELECT comic_id, ROW_NUMBER() OVER (ORDER BY sort_key, comic_id DESC) AS new_key
  FROM comic_order_index
)
UPDATE comic_order_index
SET sort_key = (SELECT new_key FROM ranked WHERE ranked.comic_id = comic_order_index.comic_id);
```

**Q3: 如何查看下载失败原因？**
```sql
SELECT comic_id, last_error, retry_count 
FROM download_tasks 
WHERE status = 'failed'
ORDER BY updated_at DESC;
```

**Q4: 如何手动触发任务对账？**
```bash
# 重启应用即可自动触发
systemctl restart comic-go
```
